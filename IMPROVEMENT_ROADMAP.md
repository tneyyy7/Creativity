# 🚀 Creativity AI — План улучшений (Improvement Roadmap)

> Дата составления: 2026-06-04
> Стек: React 18 + Vite + Supabase + Tailwind + framer-motion + i18next
> ~17 000 строк фронтенда, безопасность на RLS, без бэкенд-фреймворка

---

## 📊 Анализ проекта

### Что это сейчас
Соцсеть для художников/креаторов. Гибрид **Instagram + Behance/ArtStation/DeviantArt + Pinterest**.

### Уже реализовано (сильная база)
- Аутентификация (email/пароль, OAuth, сброс пароля), PWA, push через OneSignal
- Лента (`Explore`): подписки (хронология) + Explore с поиском/категориями/тегами/сортировкой
- Галерея работ (загрузка, конвертация HEIC, черновики/готовые), коллекции
- Посты: лайки, комментарии с ветками, теги, просмотры, закладки
- Stories (24 ч, лайки, просмотры, **видео** поддерживается)
- Мессенджер (2683 строки): личные + групповые чаты, реакции, ответы, редактирование, темы чата, mute, pin, скрытие, read receipts, presence, кастомные эмодзи
- Профили (публичные/личные), подписчики/подписки, друзья (заявки)
- Ранги (10 уровней по числу работ), продуктивность, дашборд
- Монетизация: Stripe Pro ($4.99/мес, годовой), Pro-фичи: аналитика, рамки аватара, цвет ника, кастом-эмодзи
- Liquid-glass дизайн, 3 языка (en, ru, it)

### Главные структурные проблемы

| Проблема | Где | Влияние |
|---|---|---|
| **Нет серверной пагинации** — `.range()` не используется нигде. Лента/Explore грузят ВСЕ работы и режут/сортируют на клиенте | `fetchExplorePaintings`, `fetchFeedPaintings` | 🔴 Не масштабируется дальше пары сотен постов |
| **Нет модерации** — ни жалоб, ни блокировки, ни NSFW-флагов | везде | 🔴 Блокер для роста и сторов |
| **Лента без алгоритма** — только хронология подписок + сортировка по лайкам | `Explore.jsx` | 🔴 Главный двигатель удержания отсутствует |
| **Нет шеринга/репостов** — нет вирусной петли роста | — | 🔴 Нет органического роста |
| `Messages.jsx` (2683) и `supabase.js` (2340) — монолиты | — | 🟡 Тех-долг |
| **Нет тестов, error-tracking (Sentry), product-аналитики** | — | 🟡 Слепые зоны в проде |
| **Изображения без оптимизации** — нет thumbnails/responsive/CDN-трансформов | загрузка | 🟡 Медленная лента, дорогой трафик |
| Мусор в репозитории: `.MP4`, скриншоты, `check_*.js`, россыпь `.sql` в корне | корень | 🟢 Гигиена |

## 🎯 План улучшений по фазам

### 🔴 ФАЗА 1 — Критично (фундамент для роста). 2–4 недели

#### 1.1 Серверная пагинация (ЗАМЕНИТЬ выборку «всё сразу»)
**Проблема:** `fetchExplorePaintings`/`fetchFeedPaintings` тянут все записи и сортируют в JS. Кнопка «Load more» лишь режет уже загруженный массив (`supabase.js:1550`, `Explore.jsx:163`).

**Что сделать:**
- Перевести на `.range(from, to)` с серверной сортировкой (`order` в запросе, не в JS).
- Заменить кнопку «Load more» на **бесконечный скролл через `IntersectionObserver`**.
- Денормализовать `likes_count`/`comments_count` в таблицу `paintings` через триггеры (есть `fix_analytics_counters.sql` — доделать), чтобы сортировать по популярности на сервере без `count`-агрегаций.

#### 1.2 Модерация и безопасность (ДОБАВИТЬ — блокер)
- Таблицы `reports` (жалобы на пост/юзера/коммент) и `blocked_users`.
- Кнопки «Пожаловаться» и «Заблокировать» в `PostViewerModal`, `PublicProfile`, мессенджере.
- Фильтрация заблокированных в лентах и поиске (RLS-политики + клиентский фильтр).
- Флаг `is_nsfw`/`is_sensitive` на работе + размытие в Explore (для арт-сообщества обязательно).

#### 1.3 Базовая наблюдаемость (ДОБАВИТЬ)
- **Sentry** (или аналог) для ошибок в проде.
- **Product-аналитика** (PostHog self-host / Plausible) — воронки регистрации, retention, использование фич.
- Убрать оставшиеся `console.log` из прода (`supabase.js` — 7, и др.).

### 🟠 ФАЗА 2 — Двигатели роста и вовлечения. 1–2 месяца

#### 2.1 Алгоритмическая лента «For You» (ДОБАВИТЬ)
Сейчас лента подписок чисто хронологическая, fallback — топ по числу работ. Самый слабый момент для соцсети.
- Создать **Supabase RPC / Edge Function** с ранжированием: свежесть × вовлечённость × близость интересов (категории/теги, которые юзер лайкает) × немного случайности.
- Вкладки: **«For You» (алгоритм)** / «Подписки» (хронология) / «Explore».
- Прямо поднимает время в приложении и retention.

#### 2.2 Репост / Share (ДОБАВИТЬ — вирусность)
- Кнопка «Поделиться»: внешний share (Web Share API) + внутренний репост в свой профиль/в личку.
- Открытые OG-теги для красивых превью при шеринге ссылок (сейчас `index.html` статичный — нужны динамические OG через Edge Function/мета).

#### 2.3 Видео-посты и карусели (ДОБАВИТЬ — форматы контента)
- **Видео в постах** (не только Stories) — для арта это таймлапсы процесса, киллер-формат. Инфраструктура частично есть (`isVideo()` в StoriesViewer).
- **Карусели** (мульти-изображение в одном посте) — вариации/этапы работы. Галерея уже принимает `multiple` файлов (`Gallery.jsx:256`) — расширить в полноценный пост-проект.

#### 2.4 Онбординг + рост (ДОБАВИТЬ)
- Онбординг-флоу при регистрации: выбор специализации + интересов (категории) → сразу подсунуть 5–10 авторов на подписку (иначе у новичка пустая лента).
- Реферальная программа / инвайты (бонус Pro за приглашённых).
- Email-дайджест и «вас лайкнули/прокомментировали» (расширить нынешний OneSignal-push на email-канал).

#### 2.5 Хэштеги как первоклассная сущность (ПОДПРАВИТЬ)
- Страница тега (`/tag/:name`) со своей лентой.
- Подписка на тег → попадает в For-You.
- @mentions в комментариях и постах с уведомлением.

### 🟡 ФАЗА 3 — Монетизация и удержание креаторов. 1–2 месяца

#### 3.1 Креатор-экономика (ДОБАВИТЬ — это деньги)
Сейчас монетизация только «купи себе Pro». Арт-сообщества живут на:
- **Комиссии/заказы** — статус «open for commissions» + форма заявки/прайс на профиле.
- **Чаевые (tips)** через Stripe — разовый донат автору.
- **Платные посты/коллекции** (туториалы, исходники, кисти) — Stripe-инфраструктура уже есть (`create_stripe_checkout`, webhook).
- Превращает площадку из «ещё одного Pro» в инструмент заработка → удерживает топ-креаторов.

#### 3.2 Профиль как портфолио (ПОДПРАВИТЬ)
- Закреплённые посты, link-in-bio, соц-ссылки, кастомный порядок работ, обложка профиля.
- Заимствование у Behance/ArtStation, повышает ценность профиля.

### 🟢 ФАЗА 4 — Технический долг и качество. Параллельно

#### 4.1 Рефакторинг монолитов (ЗАМЕНИТЬ структуру)
- Разбить `Messages.jsx` (2683) на: `ChatList`, `ChatWindow`, `MessageBubble`, `MessageComposer`, хуки `useChat`/`useRealtimePresence`.
- Разбить `supabase.js` (2340) на доменные модули: `api/posts.js`, `api/chat.js`, `api/social.js`, `api/stories.js`, `api/profile.js`.

#### 4.2 Оптимизация изображений (ДОБАВИТЬ)
- Генерация thumbnails + `srcset`/responsive через Supabase Image Transformations или CDN (imgproxy/Cloudflare). Сейчас в ленту грузятся полноразмерные оригиналы.
- `blurhash`/LQIP-плейсхолдеры вместо серого фона.

#### 4.3 Гигиена репозитория (ПОДПРАВИТЬ)
- Убрать из гита: `IMG_*.MP4`, `Screenshot*.png`, `creativitylogo.png` (1.9 МБ), `check_*.js`, `test_*.js` → в `.gitignore`/отдельную папку.
- Консолидировать россыпь `.sql` из корня в `supabase/migrations/` (единый источник схемы).

#### 4.4 Тесты (ДОБАВИТЬ)
- Vitest + React Testing Library на ключевые потоки: auth, отправка сообщения, лайк/закладка, загрузка работы.
- E2E (Playwright) на критичный путь: регистрация → загрузка работы → лайк.

#### 4.5 i18n (ПОДПРАВИТЬ)
- Сейчас 3 языка, но в коде много hardcoded строк («Saved», «Following», «Follow» в `Explore.jsx`, «Untitled Work» в Dashboard). Вынести в i18n.
- Странный набор языков (en/ru/it) — определись с целевыми рынками; испанский/португальский дадут больше арт-аудитории, чем итальянский.

## 🗺 Рекомендованная последовательность

```
Сейчас → Фаза 1 (пагинация + модерация + Sentry)        ← без этого нельзя расти
   ↓
Фаза 2 (For-You лента + репост + видео/карусели + онбординг)  ← рост и вовлечение
   ↓
Фаза 3 (комиссии + чаевые + портфолио)                  ← деньги и удержание креаторов
   ↓
Фаза 4 (рефакторинг + тесты + изображения)              ← параллельно, постоянно
```

## 💡 Самое важное в одном предложении

Приложение функционально богатое (мессенджер и Pro-фичи на уровне взрослых продуктов), но ему не хватает трёх вещей, без которых соцсеть не растёт — **алгоритмической ленты, механики шеринга/вирусности и серверной пагинации**. Начинать нужно с них.
